Migration Wix → GitHub → Vercel

De Wix à votre propre code.

Guide pas-à-pas pour héberger www.nexperio.com sur Vercel à partir du repo GitHub nexperio/nexperio. Compter 1 à 2 heures.

Vue d'ensemble

  1. Cloner le repo et tester en local
  2. Configurer Supabase (base de données pour le blog)
  3. Pousser sur GitHub (déjà fait — le repo existe)
  4. Déployer sur Vercel
  5. Connecter le domaine nexperio.com
  6. Couper Wix proprement

1 · Cloner et tester en local

01

Ouvrez un terminal, placez-vous où vous voulez (ex. ~/Sites) et clonez :

git clone https://github.com/nexperio/nexperio.git
cd nexperio
npm install

Si npm n'est pas installé : nodejs.org (version LTS).

2 · Configurer Supabase

02.A

Créez un compte gratuit sur supabase.com, puis un nouveau projet en région Europe (Frankfurt).

02.B

Dans SQL Editor, collez le contenu de supabase/schema.sql et cliquez Run. Cela crée la table blog_posts.

02.C

Allez dans Settings → API. Le projet est déjà créé : récupérez ces valeurs (ou utilisez celles ci-dessous, déjà préparées pour vous) :

  • Project URL → https://qarqnjzixkkunhdzgule.supabase.co
  • anon public key (alias "publishable key") → sb_publishable_3xuBGegF0JxWO8tnrQoXkA_tSgoh1TO
  • service_role key → à garder secrète, uniquement côté serveur

Dupliquez .env.example en .env.local (déjà fourni dans ce projet) :

cp .env.example .env.local

Le fichier .env.example contient déjà l'URL et la publishable key. Ajoutez la service_role et la clé Resend (cf. étape 6).

⚠️ Sécurité — la publishable key est faite pour être exposée côté client. La service_role key ne doit jamais apparaître dans du code envoyé au navigateur (pas de préfixe NEXT_PUBLIC_) ni être commitée.

Testez en local :

npm run dev
# → http://localhost:3000

3 · Pousser sur GitHub

Votre repo github.com/nexperio/nexperio existe déjà. Pour synchroniser une modif locale :

git add .
git commit -m "feat: ma modification"
git push origin main
Astuce — chaque git push redéploie automatiquement sur Vercel une fois la connexion faite (étape suivante).

4 · Déployer sur Vercel

04.A

Créez un compte sur vercel.com avec votre compte GitHub.

04.B

New Project → importez nexperio/nexperio.

Vercel détecte automatiquement Next.js. Ne touchez pas aux build settings.

04.C

Dans Environment Variables, ajoutez les deux clés Supabase :

  • NEXT_PUBLIC_SUPABASE_URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY

Cochez Production + Preview + Development.

04.D

Cliquez Deploy. ~2 minutes plus tard : URL automatique nexperio.vercel.app.

5 · Connecter nexperio.com

05.A

Dans Vercel → projet → Settings → Domains. Ajoutez :

  • nexperio.com
  • www.nexperio.com
05.B

Vercel vous donne deux enregistrements DNS à configurer chez votre registrar (OVH, Gandi, GoDaddy, etc.) :

Type   Nom    Valeur
A      @      76.76.21.21
CNAME  www    cname.vercel-dns.com

Important : tant que Wix gère le DNS, le site pointe encore vers Wix. Il faut soit :

  • Option A — récupérer la gestion DNS chez votre registrar (recommandé)
  • Option B — modifier le DNS directement dans Wix (Settings → Domains → Advanced → DNS Records)
05.C

Une fois les DNS propagés (15 min – 24 h), Vercel provisionne le SSL automatiquement (Let's Encrypt). Le cadenas vert apparaît tout seul.

6 · Resend pour le formulaire de RDV

06.A

Créez un compte sur resend.com. Plan gratuit : 3 000 emails/mois, suffisant largement.

06.B

Dans Resend → Domains, ajoutez nexperio.com. Resend vous donne 3 enregistrements DNS (SPF, DKIM, MX) à ajouter chez votre registrar — comme pour Vercel à l'étape 5.

Une fois validé (15 min – quelques heures), vous pouvez envoyer depuis noreply@nexperio.com.

06.C

Dans Resend → API Keys, créez une clé "Production" (full access) et copiez-la.

06.D

Sur Vercel → projet → Settings → Environment Variables, ajoutez :

RESEND_API_KEY=re_xxxxxxxxxxxx

Puis installez le SDK dans votre projet Next.js :

npm install resend
06.E

Le fichier api/send-rdv.ts est déjà prêt dans ce projet. Copiez-le dans votre dépôt Next.js à l'emplacement :

app/api/send-rdv/route.ts

Le formulaire de la page RDV envoie déjà POST /api/send-rdv avec les données du patient, déclenche un email vers ag@nexperio.com, puis redirige vers Calendly automatiquement.

06.F

Test : remplissez le formulaire en production, vérifiez la réception sur ag@nexperio.com et la redirection vers Calendly.

7 · Couper Wix proprement

  1. Vérifiez que https://www.nexperio.com répond bien depuis Vercel (pas de page Wix qui s'affiche)
  2. Testez toutes les pages, le formulaire de RDV, le blog
  3. Dans Wix : Settings → Subscriptions → Cancel plan
  4. Conservez l'export de votre contenu Wix (au cas où) avant de fermer le compte
Conseil migration — gardez Wix actif pendant 1 à 2 semaines après bascule, le temps de vérifier que tout fonctionne. Coupez seulement après.

Ajouter un article de blog

Via Supabase Table Editor → blog_posts → Insert row.

Ou en SQL :

INSERT INTO blog_posts (title, slug, excerpt, content, first_published_at, featured, category_names)
VALUES (
  'Mon titre',
  'mon-slug-unique',
  'Résumé court...',
  '<p>Contenu HTML...</p>',
  now(),
  false,
  ARRAY['Catégorie']
);

En cas de pépin

  • Build qui plante sur Vercel → onglet Deployments → cliquez sur le dernier build → lisez les logs
  • SSL "non sécurisé" → patientez, ça se règle dans les 24 h après propagation DNS
  • Le blog est vide → vérifiez que les variables d'env Supabase sont bien dans Vercel
  • Le DNS ne propage pas → testez avec dnschecker.org
